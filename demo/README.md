## The Talk Structure (25 Minutes)

**Why this structure works:**
It mirrors how engineers actually solve problems - not with theory, but with a painful moment that sparks an insight that leads to building something real.

```
HOOK (2 min)
  ↓
PROBLEM (3 min)
  ↓
INSIGHT (2 min)
  ↓
DEMO (12 min) ← 48% of talk time
  ↓
IMPACT (3 min)
  ↓
CLOSE (2 min)
  ↓
Q&A (5 min)
```

## How to Prepare

### 3 Weeks Before:
1. **Build the slide deck** (use slide_deck.md as blueprint)
   - 7 slides total
   - Minimal text, maximum visuals
   - Test on projector or large screen

2. **Record the backup demo video**
   - Same environment, same board, same project
   - 12 minutes max
   - Have it on a second laptop ready to go

### 2 Weeks Before:
3. **Practice the Pomodoro demo 10+ times**
   - Run it live until you know where every button is
   - Fix the top 3 failure points
   - Test Cato visuals for projector readiness

4. **Practice the full talk out loud 5+ times**
   - Time yourself (target 23-25 minutes)
   - Record yourself once and watch it
   - Practice the pause moment until it's muscle memory

### 1 Week Before:
5. **Give the talk to a test audience**
   - Family member, friend, colleague
   - Get feedback on pacing and clarity
   - Adjust based on what confused them

6. **Prep hard questions**
   - Know the 5 hardest questions cold (see speaker_notes.md)
   - Practice answering out loud
   - Don't memorize - internalize

### Day Before:
7. **Complete rest**
   - No changes, no fixes, no new ideas
   - Be with your family
   - Sleep

### Day Of:
8. **Pre-talk checklist** (see speaker_notes.md)
   - Test tech setup 30 min before
   - Run demo one last time
   - Deep breath. You've got this.

## The Moments That Matter Most

### 1. The Opening Hook (First 30 seconds)
> "Six months ago, I was watching Claude grind through a task that was taking way too long. I thought: 'If one agent takes an hour, shouldn't two agents take thirty minutes?'"

**Why it works:** Personal, relatable, sets up the tension immediately.

### 2. The Insight Pause (Slide 3)
> "What if agents never talked to each other at all?"

**[3 seconds of silence]**

**Why it works:** The pause makes the insight land. Resist the urge to fill the silence.

### 3. The Demo Pause Moment (Minute 8-9)
Stop mid-execution. Show the board state. Say:
> "This is everything any agent in this system knows. Complete context. No direct communication needed."

**[5 seconds of silence - let them read the screen]**

**Why it works:** This is your mic drop. The audience sees the philosophy made concrete.

## What Makes This Talk Different

**Most framework talks:**
- Start with theory
- Show architecture diagrams
- Demo at the end (if at all)
- Assume audience cares about the tech

**This talk:**
- Starts with a human moment
- Shows the pain first
- Demo is 48% of the talk
- Connects to audience's real problems
- Ends with "try it and tell me what breaks" (invites collaboration)

**Why it works:** Engineers don't care about frameworks. They care about problems they've felt and solutions that actually work.
